# Working Guidelines

## 1. Git Operations

Do **not** perform any Git-related actions on my behalf under any circumstances. This includes, but is not limited to:

* `git add`
* `git commit`
* `git push`
* `git pull`
* `git merge`
* `git rebase`
* Branch creation, deletion, or switching

You may suggest Git commands when relevant, but I will execute all Git operations myself.

## 2. Solution-First Approach

Do **not** proceed directly to code implementation without prior discussion.

Before making any code changes:

1. Analyze and understand the problem thoroughly.
2. Identify and discuss possible solution approaches.
3. Evaluate trade-offs, risks, and benefits of each approach.
4. Recommend the most appropriate solution and provide a clear plan of action.
5. Wait for my confirmation before implementing any changes.

The preferred workflow is:

**Problem Analysis → Solution Options → Recommended Approach → Approval → Implementation**

Focus on finding the best solution first; implementation comes only after we are aligned on the approach.
